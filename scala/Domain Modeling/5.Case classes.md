case class常用于定义不可变的数据结构
```scala
case class Person(name: String, relation: String)

val christina = Person("Christina", "niece")
```
默认情况下case class的成员是public且immutable的

实际上使用case class时，scala编译器会自动创建一个定义了apply方法的半生对象
```scala
object Person {
  def apply(name: String, relation: String): Person =
    new Person(name, relation)
}
```
进行实例化时相当于：
```scala
val christina = Person.apply("Christina", "niece")
```
所以说实例化case class时不需要new

## 模式匹配
case class可以与match结合使用
```scala
case class Person(name: String)

def speak(p: Person) = p match {
  case Person(name) if name == "Fred" => println(s"$name says, Yubba dubba doo")
  case Person(name) if name == "Bam Bam" => println(s"$name says, Bam bam!")
  case _ => println("Watch the Flintstones!")
}

speak(Person("Fred"))      // "Fred says, Yubba dubba doo"
speak(Person("Bam Bam"))   // "Bam Bam says, Bam bam!"
```

## 其他scala给出的case class方法
```scala
// `equals` and `hashCode` methods generated for you
val hannah = Person("Hannah", "niece")
christina == hannah       // false

// `toString` method
println(christina)        // Person(Christina,niece)

// built-in `copy` method
case class BaseballTeam(name: String, lastWorldSeriesWin: Int)
val cubs1908 = BaseballTeam("Chicago Cubs", 1908)
val cubs2016 = cubs1908.copy(lastWorldSeriesWin = 2016)
// result:
// cubs2016: BaseballTeam = BaseballTeam(Chicago Cubs,2016)
```
其中copy方法是对case class实例进行复制并修改的主要途径。

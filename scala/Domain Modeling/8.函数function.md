scala的method可以理解为一段有返回值或者副作用的statement，而scala的function本身就是一个值。使用体验上，scala的method类似c的函数，而scala的function类似c的函数指针。

scala的function本质上是一个FunctionN trait混入匿名类并实例化得到的实例。
scala定义了23个独立的trait：Function0, ... ,Function22
每个trait都只声明了apply函数，并且规定了apply函数的参数量,这导致scala的function最多接受22个形参。
定义一个scala函数时会根据设定的参数量选择对应的trait,并混入一个匿名函数，再实例化。
这段思路展开写是这样的：
```scala
//REPL
scala> val addTwoInt:(Int,Int)=>Int =
     |  new Function2[Int,Int,Int] {
     |  override def apply(a:Int,b:Int):Int = {a+b}
     | }
val addTwoInt: (Int, Int) => Int = <function2>

scala> val x=addTwoInt(1,2)
val x: Int = 3
```

scala定义函数的标准语法如下
```scala
val <name>: (<ArgTypes>) => <ReturnType> = (<params>) => <expression>
```

实际上scala 2.12引入了JVM lambda做优化，但由于类型系统的限制，22个形参的上限仍然存在。
```scala
//REPL
scala> val addTwoInt:(Int,Int)=>(Int) = (a,b)=>{a+b}
val addTwoInt: (Int, Int) => Int = $Lambda/0x00007f4150551220@69a90b81

scala> val x=addTwoInt(1,2)
val x: Int = 3
```

*从scala3开始，function解除了形参数量的限制，会在底层把形参包装成tuple。*

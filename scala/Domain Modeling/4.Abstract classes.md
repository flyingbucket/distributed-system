当你要定义一个类，但其中包含**抽象成员（abstract members）**时，你可以选择使用 **trait** 或 **abstract class**。  
在大多数情况下，Scala 推荐使用 **trait**，但有两个场景更适合使用 **abstract class**：
### **需要主构造器参数时（constructor arguments）**  

scala2中trait **不能** 直接带有构造参数，而抽象类可以。例如：

```scala
abstract class Animal(val name: String) // OK
trait AnimalTrait(val name: String)  // 错误，trait 不能带参数
```
因此，如果你需要在父类中定义构造参数（比如名称、ID、配置等），就应该用 `abstract class`

*ps.scala3的trait支持构造参数，更加推荐使用trait而不是abstract class*
### **需要与 Java 代码交互时（Java interoperability）**  
Java 不能直接继承或实现 Scala 的 `trait`（尤其是带有实现体的 trait），但可以继承抽象类。

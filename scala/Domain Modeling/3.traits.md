## trait的基本用法
trait 的主要作用，就是把**多个类中可能都会用到的通用方法或行为**抽取出来，放在一个统一的地方定义，方便**复用和组合**。

trait中可以声明抽象方法，也可以给出方法的具体实现。

```scala
trait HasLegs {
  def numLegs: Int //声明无参数方法
  def walk(): Unit //声明抽象方法
  def stop() = println("Stopped walking") //定义具体方法
}
```
trait还可以声明无参数方法，无参数方法就是不接受任何参数的方法。访问一个对象的无参数方法时，语法和访问属性是一样的，不需要加括号，无参数方法的返回值也是该字段对应的“属性”值，实际使用体验和访问对象属性是一致的。

```scala
//声明无参数方法
trait Employee {
  def id: Int
  def firstName: String
  def lastName: String
}
```

## trait的抽象接口在类中的实现
普通方法在类中的实现很简单，抽象方法直接给出实现，具体方法用override重写。
无参数方法的实现稍微复杂，这也是scala的一大特性。
```scala
trait Employee {
  def id: Int            // 抽象无参数方法
  def name: String       // 抽象无参数方法
}
```
### 用val实现
```scala
class Manager(val id: Int, val name: String) extends Employee
```
在类的住构造器中声明`val id.val name`。在实例化时传入给id和name的值会被直接设置为trait中无参数方法的返回值。

需要注意的是，该属性必须声明为val,因为trait中的无参数方法是只读的。
### 用def实现
```scala
class Engineer(val id: Int, first: String, last: String) extends Employee {
  def name: String = s"$first $last"
}
```
